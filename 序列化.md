# System.Text.Json 序列化配置详解文档

> 适用版本：.NET 6 / .NET 7 / .NET 8
>  主要对象：`System.Text.Json.JsonSerializerOptions`
>  范围说明：**仅聚焦“序列化（Serialize）行为”**，反序列化只在必要时提示

------

## 1. 概述

`System.Text.Json` 是 .NET 官方内置的 JSON 序列化组件，
 其核心特点是：

- **默认行为严格**
- **高性能、低内存**
- **高度依赖配置项控制输出行为**

所有序列化相关行为，几乎都通过 `JsonSerializerOptions` 控制。

```
JsonSerializer.Serialize(object value, JsonSerializerOptions options);
```

------

## 2. 最常用核心配置项（必读）

### 2.1 属性命名策略（PropertyNamingPolicy）

#### 作用

控制 **C# 属性名 → JSON 字段名** 的转换规则。

#### 配置方式

```
PropertyNamingPolicy = JsonNamingPolicy.CamelCase;
```

#### 示例

```
public class User
{
    public string UserName { get; set; }
}
```

| 配置      | 输出                    |
| --------- | ----------------------- |
| 默认      | `{ "UserName": "Tom" }` |
| CamelCase | `{ "userName": "Tom" }` |

✅ **Web API 强烈建议开启**

------

### 2.2 忽略 null / 默认值（DefaultIgnoreCondition）

#### 配置方式

```
DefaultIgnoreCondition = JsonIgnoreCondition.WhenWritingNull;
```

#### 可选值说明

| 值                   | 行为                           |
| -------------------- | ------------------------------ |
| `Never`              | 不忽略（默认）                 |
| `WhenWritingNull`    | 忽略 `null`                    |
| `WhenWritingDefault` | 忽略默认值（0 / false / null） |

⚠️ **注意**

- `WhenWritingDefault` 可能导致字段“消失”
- DTO / API 返回值中慎用

------

### 2.3 是否格式化输出（WriteIndented）

```
WriteIndented = true;
```

| 场景            | 建议            |
| --------------- | --------------- |
| 调试 / 日志     | `true`          |
| 生产 / API 返回 | `false`（默认） |

------

### 2.4 枚举序列化方式（Converters）

#### 默认行为

```
"status": 1
```

#### 转为字符串

```
Converters.Add(new JsonStringEnumConverter());
"status": "Enabled"
```

✅ **前后端交互推荐使用字符串枚举**

------

## 3. Dictionary / Key 相关配置

### 3.1 Dictionary Key 命名策略（DictionaryKeyPolicy）

```
DictionaryKeyPolicy = JsonNamingPolicy.CamelCase;
var dict = new Dictionary<string, string>
{
    ["UserName"] = "Tom"
};
```

输出：

```
{
  "userName": "Tom"
}
```

⚠️ 不设置时，Dictionary Key **不会**应用 `PropertyNamingPolicy`

------

## 4. 对象引用与循环处理（非常重要）

### 4.1 循环引用问题

```
class Node
{
    public Node Parent { get; set; }
}
```

默认行为：**直接抛异常**

------

### 4.2 忽略循环引用（常用）

```
ReferenceHandler = ReferenceHandler.IgnoreCycles;
```

- 遇到循环引用直接忽略
- 最常见、最安全

------

### 4.3 保留引用关系（高级）

```
ReferenceHandler = ReferenceHandler.Preserve;
```

输出示例：

```
{
  "$id": "1",
  "parent": {
    "$ref": "1"
  }
}
```

⚠️ 前端必须支持 `$id / $ref`，否则不建议使用

------

## 5. 深度控制（防御性配置）

### 5.1 最大深度（MaxDepth）

```
MaxDepth = 64; // 默认 64
```

作用：

- 防止对象图过深
- 避免 StackOverflow / 性能问题

------

## 6. 字段 & 访问级别相关

### 6.1 是否包含字段（IncludeFields）

```
IncludeFields = true;
```

默认行为：

- 只序列化 **属性**
- `public field` 会被忽略

启用后：

- `public field` 也会输出

------

### 6.2 忽略只读属性 / 字段

```
IgnoreReadOnlyProperties = true;
IgnoreReadOnlyFields = true;
```

适合：

- DTO 精简输出
- 只暴露可写数据

------

## 7. 数字与 JSON 规范控制

### 7.1 数字处理（NumberHandling）

```
NumberHandling = JsonNumberHandling.WriteAsString;
```

#### 常见选项

| 选项                              | 说明                       |
| --------------------------------- | -------------------------- |
| `AllowReadingFromString`          | `"18"` → `int`（反序列化） |
| `WriteAsString`                   | 数字写成字符串             |
| `AllowNamedFloatingPointLiterals` | NaN / Infinity             |

⚠️ `WriteAsString` 会改变前端数据类型

------

## 8. 编码与安全（Web 场景）

### 8.1 JSON Encoder

```
Encoder = JavaScriptEncoder.UnsafeRelaxedJsonEscaping;
```

#### 默认行为

```
"<" → "\u003c"
```

#### Unsafe 行为

```
"<" → "<"
```

⚠️ **注意**

- 返回 HTML / 用户输入内容时需谨慎
- 防止 XSS 风险

------

## 9. 常见完整推荐配置（生产模板）

```
var options = new JsonSerializerOptions
{
    PropertyNamingPolicy = JsonNamingPolicy.CamelCase,
    DictionaryKeyPolicy = JsonNamingPolicy.CamelCase,
    DefaultIgnoreCondition = JsonIgnoreCondition.WhenWritingNull,
    WriteIndented = false,
    ReferenceHandler = ReferenceHandler.IgnoreCycles,
    MaxDepth = 64,
    Encoder = JavaScriptEncoder.UnsafeRelaxedJsonEscaping
};

options.Converters.Add(new JsonStringEnumConverter());
```

------

## 10. ASP.NET Core 全局配置示例

```
builder.Services.AddControllers()
    .AddJsonOptions(options =>
    {
        var json = options.JsonSerializerOptions;

        json.PropertyNamingPolicy = JsonNamingPolicy.CamelCase;
        json.DictionaryKeyPolicy = JsonNamingPolicy.CamelCase;
        json.DefaultIgnoreCondition = JsonIgnoreCondition.WhenWritingNull;
        json.ReferenceHandler = ReferenceHandler.IgnoreCycles;
        json.Converters.Add(new JsonStringEnumConverter());
    });
```

------

## 11. 常见误区总结

| 问题                 | 原因                            |
| -------------------- | ------------------------------- |
| 字段名没变 camelCase | 忘了设置 `PropertyNamingPolicy` |
| 循环引用异常         | 没配置 `ReferenceHandler`       |
| 数字变字符串         | 使用了 `WriteAsString`          |
| Dictionary Key 不变  | 没配置 `DictionaryKeyPolicy`    |
| 前端字段缺失         | 使用了 `WhenWritingDefault`     |

------

## 12. 总结

> **System.Text.Json 的核心使用原则：**
>
> - 行为集中在 `JsonSerializerOptions`
> - 默认严格，需显式配置
> - 推荐统一全局配置
> - DTO / API 返回模型要“为序列化设计”





# System.Text.Json 反序列化配置详解

> 适用版本：.NET 6 / .NET 7 / .NET 8  
> 组件：System.Text.Json  
> 主要对象：JsonSerializerOptions  
> 说明：本文档 **仅聚焦“反序列化（Deserialize）行为”**

---

## 1. 概述

System.Text.Json 在**反序列化**时默认行为非常严格，  
相比 Newtonsoft.Json：

- 类型必须匹配
- JSON 结构必须规范
- 很多“宽容行为”需要显式开启

```csharp
JsonSerializer.Deserialize<T>(string json, JsonSerializerOptions options);
```

## 2. 最常用反序列化配置（必读）

### 2.1 属性名是否忽略大小写（PropertyNameCaseInsensitive）

#### 默认行为

- **区分大小写**

```
{ "userName": "Tom" }
public string UserName { get; set; } // ❌ 默认匹配失败
```

------

#### 推荐配置

```
PropertyNameCaseInsensitive = true;
```

✅ **Web API / 对接前端几乎必配**

------

## 3. 数字与类型容错（非常关键）

### 3.1 允许数字从字符串读取（NumberHandling）

#### JSON 示例

```
{ "age": "18" }
```

#### 默认行为

```
public int Age { get; set; } // ❌ 抛异常
```

------

#### 推荐配置

```
NumberHandling = JsonNumberHandling.AllowReadingFromString;
```

| 选项                   | 说明                     |
| ---------------------- | ------------------------ |
| AllowReadingFromString | `"18"` → `int`           |
| WriteAsString          | 数字写成字符串（序列化） |

⚠️ 这是 **System.Text.Json 与 Newtonsoft 最大差异之一**

------

## 4. 枚举反序列化行为

### 4.1 枚举字符串反序列化

#### JSON

```
{ "status": "Enabled" }
```

#### 默认行为

```
public Status Status { get; set; } // ❌ 抛异常
```

------

#### 正确配置

```
Converters.Add(new JsonStringEnumConverter());
```

支持：

- `"Enabled"`
- `"enabled"`（大小写不敏感）

------

## 5. JSON 规范宽容度配置

### 5.1 是否允许尾随逗号（AllowTrailingCommas）

#### JSON 示例（非标准）

```
{
  "name": "Tom",
}
```

#### 配置

```
AllowTrailingCommas = true;
```

| 默认 | false（抛异常） |
| ---- | --------------- |
|      |                 |

------

### 5.2 是否允许注释（ReadCommentHandling）

#### JSON 示例

```
{
  // user name
  "name": "Tom"
}
```

#### 配置

```
ReadCommentHandling = JsonCommentHandling.Skip;
```

| 值       | 说明               |
| -------- | ------------------ |
| Disallow | 不允许（默认）     |
| Skip     | 忽略注释           |
| Allow    | 读取注释（很少用） |

------

## 6. 未知字段处理

### 6.1 JSON 多余字段

```
{
  "name": "Tom",
  "extra": "value"
}
public class User
{
    public string Name { get; set; }
}
```

#### 默认行为

- ✅ **自动忽略**
- ❌ 不会抛异常

⚠️ System.Text.Json **不支持**“遇到未知字段就报错”的配置

------

## 7. 构造函数与只读属性

### 7.1 使用构造函数反序列化

```
public class User
{
    public string Name { get; }

    public User(string name)
    {
        Name = name;
    }
}
```

#### 要求

- 构造函数参数名 **必须匹配 JSON 属性名**
- 支持 `[JsonConstructor]`

```
[JsonConstructor]
public User(string name) { ... }
```

------

### 7.2 私有 setter 行为

```
public string Name { get; private set; }
```

| 行为         | 说明     |
| ------------ | -------- |
| 默认         | ❌ 不赋值 |
| 使用构造函数 | ✅ 支持   |

------

## 8. 字段反序列化

### 8.1 是否允许字段赋值（IncludeFields）

```
IncludeFields = true;
```

| 默认 | false |
| ---- | ----- |
|      |       |

仅对 `public field` 生效

------

## 9. 深度与安全控制

### 9.1 最大深度（MaxDepth）

```
MaxDepth = 64;
```

- 防止恶意 JSON
- 防止栈溢出

------

## 10. 循环引用（反序列化）

### 10.1 ReferenceHandler.Preserve

```
ReferenceHandler = ReferenceHandler.Preserve;
```

支持解析：

```
{
  "$id": "1",
  "parent": { "$ref": "1" }
}
```

⚠️ 仅在你明确知道 JSON 使用 `$id / $ref` 时启用

------

## 11. 常见反序列化推荐配置模板

```
var options = new JsonSerializerOptions
{
    PropertyNameCaseInsensitive = true,
    NumberHandling = JsonNumberHandling.AllowReadingFromString,
    AllowTrailingCommas = true,
    ReadCommentHandling = JsonCommentHandling.Skip,
    MaxDepth = 64
};

options.Converters.Add(new JsonStringEnumConverter());
```

------

## 12. ASP.NET Core 全局反序列化配置示例

```
builder.Services.AddControllers()
    .AddJsonOptions(options =>
    {
        var json = options.JsonSerializerOptions;

        json.PropertyNameCaseInsensitive = true;
        json.NumberHandling = JsonNumberHandling.AllowReadingFromString;
        json.AllowTrailingCommas = true;
        json.ReadCommentHandling = JsonCommentHandling.Skip;
        json.Converters.Add(new JsonStringEnumConverter());
    });
```

------

## 13. 常见反序列化问题速查表

| 问题                  | 原因                               |
| --------------------- | ---------------------------------- |
| JSON 字段大小写不匹配 | 未启用 PropertyNameCaseInsensitive |
| `"18"` → int 报错     | 未启用 AllowReadingFromString      |
| 枚举字符串失败        | 未加 JsonStringEnumConverter       |
| JSON 有注释报错       | 未配置 ReadCommentHandling         |
| 尾随逗号报错          | 未启用 AllowTrailingCommas         |

------

## 14. 总结

> System.Text.Json 反序列化原则：
>
> - 默认 **严格且安全**
> - 对“非标准 JSON”需要显式配置
> - 第三方接口 / 前端数据建议放宽规则
> - 构造函数优于 setter，DTO 设计很关键
